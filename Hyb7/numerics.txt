This is a comprehensive list and description of all of the Hybrid 7
numerics that can be generated by the server.  This document is designed
for client and script writers, to provide an easier way to get this
information than having to read the source code themselves.

All numerics are of the form :originating.server numeric user parameters
The parameters are described after listing the numerics.  Words in all
capital letters are items that the server will insert text.  All
substitutions are listed in order.  The default text is not listed, as
it is easily (and often) changed.  If a : is listed in the text, it is
sent explicitly by the server.

RPL_WELCOME          001  :Welcome text, containing NETWORKNAME and NICK
              Used right when a user it properly registered to the server.
RPL_YOURHOST         002  :Lists the SERVERHOST and the VERSION
              Used after registration, gives the client a brief description
              of what version the server they are using is running.
RPL_CREATED          003  :String with the STARTTIME when ircd was started
              Used after registration.
RPL_MYINFO           004  SERVERNAME VERSION USERMODES CHANMODES PARAMMODES
              Used after registration, this lists what modes are supported
              by the server.  PARAMMODES are channel modes that require
              parameters.  It is recommended that you use RPL_ISUPPORT
              rather than parsing 002/003/004.
RPL_ISUPPORT         005  LIST OF SUPPORTED TOKENS :description text
              Used to provide a list of what client/server protocol items
              are supported by this server.  This is documented better in
              doc/whats-new.txt.
RPL_REDIR            010  SERVERNAME PORT :human readable description
              Used in server side redirection to provide an alternative
              server and port for the client to connect to.  This feature
              is from IRCnet.
RPL_MAP              015  :generated map text
              Used in the optional m_map.c module to show MAP information.
              This feature is originally from Undernet.
RPL_MAPMORE          016  !!! THIS NUMERIC IS UNUSED !!!
              Reserved for future expansion
RPL_MAPEND           017  :text signifying the end of the MAP command
              Used in the optional m_map.c module to show the end of the
              MAP results.  This feature is originally from Undernet.
RPL_YOURID           020  ID KEY :description text
              Not currently used.  This will be used for persistant client
              and cookies support.  This is semi-compatible with TS4's
              cookie implementation.
